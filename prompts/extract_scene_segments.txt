You extract scene/segment summaries from a Chinese fantasy novel chapter for building RAG metadata.

Output requirements (STRICT):
- Output ONLY JSONL (one JSON object per line). No markdown fences, no commentary, no blank lines before/after.
- If no scenes are found, output nothing (empty output).

Each JSON line MUST be exactly:
{
  "chapter_number": <int>,
  "scene_title_zh": "<string>",
  "scene_summary_zh": "<string>",
  "scene_summary_vi": "<string>",
  "entities": ["<canonical_id>", ...],
  "evidences": [
    { "source_file": "<filename>", "evidence_zh": "<short Chinese quote>" }
  ]
}

Hard rules:
- canonical_id in entities MUST be taken ONLY from the provided glossary block (chapter_glossary_block).
- scene_title_zh should be a short title describing the turning point/segment in Chinese.
- scene_summary_zh and scene_summary_vi are summaries only (no full translation of the story).
- evidences must be direct quotes from the chunk (<= 220 chars each).

Glossary block (JSONL entries):
<start_of_glossary>
{chapter_glossary_block}
<end_of_glossary>

Timeline events (JSONL subset; may be empty):
<start_of_timeline>
{timeline_events_block}
<end_of_timeline>

Chapter number:
{chapter_number}

Source filename:
{source_file}

Chinese chunk to analyze:
<start_of_chunk>
{source_chunk}
<end_of_chunk>
